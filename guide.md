# Nexus Technical Specification: Hybrid SMC-AI Integration

## AI Validation Protocol
The system does not execute a trade solely on an Order Block touch. When a "Technical Signal" is generated by `core/confluence.py`, it is passed to `models/inference.py`.

1. **Feature Vector:** The system flattens the OHLCV data + SMC labels into a 5-dimensional tensor.
2. **Inference:** The GRU model calculates a `confidence_score`.
3. **Threshold:** If `confidence_score` < `settings.yaml -> confluence -> min_ai_confidence`, the signal is discarded and logged as "AI Filtered."

## Sentiment Weighting
The `SentimentAnalyzer` uses the **FinBERT** model. If the sentiment is "Highly Negative" (-0.8 or lower), Bullish Long signals are automatically suppressed, regardless of chart structure, to avoid "catching a falling knife" during news-driven crashes.

## GitHub Actions Logic
* **`ai_retrain.yml`**: Automates the evolution of the model. It ensures the weights in `models/weights/` are never older than 7 days.
